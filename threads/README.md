# Threads — one directory per question

The repo is organized by **research question**, not by artifact type.
Everything a thread produces lives in its own directory:

```
threads/<slug>/
    *.md               theory/spec docs for this question (fill Prior art
                       BEFORE code — see templates/THEORY.md)
    *.py               thread-specific models/training (imports kit/, domains/)
    experiments/       this thread's runs, one dir per run id
    archive/           this thread's dead ends (kept, marked, unlinked)
```

New threads are born from theory, not code:
`python -m src.experiment theory <kebab-slug> --thread <new_thread_slug>`

Threads die by archival: move the whole directory under `threads/<slug>` to
`theories/archive/threads/` (or mark its docs superseded) — never delete;
the ledger and git history carry the memory.

## Shared layers (what threads import)

| Tier | Package | Rule |
|---|---|---|
| 0 — universal | `kit/` | primitives that would work in any project: training loop, run artifacts, synthetic tasks, shared nn blocks |
| 1 — project | `domains/` | codebases shared across threads: `byte` (vocab/tokenizers), `rwkv` (nano core, LoRA, vocab), `g1g` (frozen 2.9B plumbing) |
| 2 — thread | `threads/<slug>/` | one question; code that no other thread should import |

If two threads need the same code, it moves UP a tier (with its tests), it
never gets copied sideways.
