# Hybrid Latent-State Language Model

**Core Hypothesis**: A model with `latent_state_update() × N` + `decode_token() × M` can outperform an equivalent token-by-token model on long-horizon reasoning tasks.

## The Idea

Current LLMs force reasoning to serialize through tokens. Each token requires expensive computation. What if we could:
1. **Think** in a cheap latent state (multiple updates per token)
2. **Speak** by decoding tokens from that state (cheap)

This is like the brain/mouth distinction: the brain does expensive computation, the mouth just vocalizes thoughts. Current LLMs use the same giant stack for both thinking and speaking.

## Architecture

```
              INPUT TOKENS
                   |
                   v
          [Sequential SSM Processing]
                   |
          For each token:
            1. Embed token
            2. Update SSM state
            3. Every `think_every` tokens: 
               do N latent thinking steps
            4. Decode state → logits
                   |
                   v
                OUTPUT
```

**Key Components**:
- **SSM**: State Space Model — recurrent update `h_t = A·h_{t-1} + B·x_t`
- **Thinking Loop**: Extra computation in latent space without consuming tokens
- **Sequential Processing**: Process tokens one-by-one (not mean pooling)
- **Unified Output**: All models produce `[batch, seq_len, vocab_size]` for fair comparison

## Models

### 1. BaselineTransformer
Standard causal transformer with attention. Reference point.

### 2. LatentSSM
Sequential token processing with periodic thinking steps.
- `latent_steps`: Number of thinking iterations (e.g., 4)
- `think_every`: How often to think (e.g., every 4 tokens)

### 3. LatentSSMDecoder
SSM + multi-token decoder. Tests if we can generate multiple tokens cheaply from one state.

## Implementation Details

### Sequential Token Processing
Unlike the initial mean-pooling approach, LatentSSM now processes tokens sequentially:
```python
for t in range(seq_len):
    # Update SSM state with current token
    for ssm in self.ssms:
        h = ssm.step(x[:,t,:], h)
    
    # Periodic latent thinking
    if self.te > 0 and (t+1) % self.te == 0:
        h = self.think(h)
    
    # Decode to logits
    logits_t = self.token_decoder(h)
```

### Thinking Loop
The `think()` method performs extra computation in latent space:
```python
def think(self, h, steps=None):
    for _ in range(steps or self.latent_steps):
        h_new = h
        for ssm in self.ssms:
            h_new = ssm.step(h_new, h_new)  # Self-recurrence
        h_new = self.ffn(h_new)
        # Gated residual
        gate = self.gate(h)
        h = gate * h_new + (1 - gate) * h
    return h
```

### Evaluation
Temperature sampling with multiple attempts:
```python
def generate(prompt, temperature=0.8, top_k=40):
    # Apply temperature
    logits = logits / temperature
    
    # Top-k filtering
```

## Troubleshooting

### P100 GPU Issues
The notebook auto-detects P100 and installs compatible PyTorch (2.3.1+cu118). If issues persist, try T4 GPU.

### Out of Memory
- Reduce `batch_size` from 32 to 16
- Reduce sequence length from 128 to 64
- Reduce model dimension from 256 to 128

### Slow Training
- Latent models are inherently slower (sequential processing)
- This is expected and part of the hypothesis test
- On GPU, should still complete in reasonable time

### QA Accuracy is 0
- This is expected for early epochs (model hasn't converged)
- After 20-30 epochs on GPU, should see non-zero accuracy
- Temperature sampling helps but model needs to learn patterns first
