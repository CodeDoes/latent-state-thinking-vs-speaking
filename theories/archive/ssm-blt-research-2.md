Deep Sequence Architectures at the Interface of Bytes and Latent States: A Unified Analysis of Token-Free Modeling, Dynamic Patching, and Structured State SpacesThe Paradigm Shift in Token-Free RepresentationThe standard architecture of modern large language models relies on a static vocabulary of subword units generated through pre-processing heuristics such as Byte-Pair Encoding (BPE). While BPE reduces the sequence length of raw text to make attention-based processing computationally feasible, it introduces systematic biases that limit model flexibility, cross-domain adaptation, and input robustness. Because these subword vocabularies are heavily optimized for dominant Western European languages and programming code, they exhibit severe morphological incompatibility when applied to low-resource or non-Latin scripts, such as Thai, Khmer, or Vietnamese. This structural mismatch leads to the "Tokenization Bottleneck," which causes disproportionately high character-to-token compression ratios, increased API costs, high generation latency, and reduced context capacity for non-Western languages.Furthermore, subword tokenizers are highly sensitive to character-level perturbations, spelling errors, and casing changes, as minor inputs can drastically alter the tokenization sequence and corrupt the underlying semantic representation.Raw Text:        t  h  e     q  u  i  c  k     b  o  o  k

To resolve these limitations, Byte Language Models (BLMs) discard tokenizers to operate directly on raw UTF-8 byte streams. By modeling a fixed, universal vocabulary of 256 byte values, BLMs achieve script agnosticism, simplify multi-modal processing, and show high resilience to character-level noise.However, direct byte-level modeling increases the sequence length by $3.8\times$ to $4\times$ compared to subword sequences, creating a severe computational bottleneck. Because standard self-attention scales quadratically, $\mathcal{O}(L^2)$, with the sequence length $L$, running raw byte-level Transformers is computationally prohibitive for long-context applications.As an intermediate step between heuristic subword tokenization and raw byte-level sequence processing, alternative segmentation techniques have emerged. One such framework is ByteSpan, an information-driven subword tokenizer that utilizes an external, frozen byte-level language model during training to identify highly predictable byte sequences and group them into variable-length spans.By learning these spans based on entropy dynamics rather than static frequency counts, ByteSpan preserves local morphological alignment while reducing sequence length. This transition from static, dictionary-based tokenization to dynamic, content-aware latent representations forms the foundation of modern high-performance byte-level architectures.Dynamic Latent Tokenization and the Byte Latent TransformerThe Byte Latent Transformer (BLT) scales byte-level modeling to parameter budgets equivalent to state-of-the-art subword models. Instead of executing expensive self-attention over every raw byte, BLT dynamically partitions the byte stream into variable-length patches.The primary computational load is handled by a large global latent Transformer operating on these patch representations, while lightweight local encoder and decoder modules manage character-level transitions.The Entropy-Based PatcherThe dynamic patching mechanism in BLT relies on information density rather than predefined lexicons. To determine patch boundaries, BLT uses a small, auxiliary byte-level autoregressive language model (the patcher, typically configured with 100M parameters, 14 layers, and a hidden dimension of 512) to compute the next-byte entropy $H(x_i)$ at each position. The entropy is calculated over the 256-symbol byte vocabulary $\mathcal{V}$ as:$$H(x_i) = -\sum_{v \in \mathcal{V}} p_e(x_i = v \mid x_{<i}) \log p_e(x_i = v \mid x_{<i})$$where $p_e(x_i = v \mid x_{<i})$ is the next-byte probability distribution predicted by the patcher.During data loading, the patcher identifies sequence boundaries by monitoring spikes in entropy. When $H(x_i)$ exceeds a pre-configured threshold $\theta_g$ (with a default value of 1.34), it indicates a transition to an unpredictable region of text, such as the start of a new word or a script change. This entropy spike triggers a new patch boundary.Predictable spans of text (low entropy) are grouped into long latent patches, whereas unpredictable spans (high entropy) are mapped to short, high-resolution patches. This dynamic partitioning focuses computational resources on high-uncertainty regions while compressing redundant spans.Hash n-Gram Embeddings and Local Cross-Attention PoolingTo capture multi-byte morphological patterns without expanding the vocabulary, BLT introduces a vocabulary-free hash $n$-gram embedding module. For each byte sequence, the model computes hash-based signatures over sliding windows of size $n \in \{3, 4, 5, 6, 7, 8\}$. These signatures are mapped via a hash function to a fixed-size embedding table with $500,002$ entries. The resulting $n$-gram embeddings are combined with the raw byte embeddings, normalized, and passed to the local encoder.The local encoder is a lightweight Transformer that processes raw byte and $n$-gram representations using a block-local causal attention mask, allowing each byte to attend only to neighboring bytes within its local region.To compress these variable-length local representations into a sequence of $M$ global latent tokens $\mathbf{T} = [\mathbf{t}_1; \mathbf{t}_2; \dots; \mathbf{t}_M] \in \mathbb{R}^{M \times d_{\text{global}}}$, BLT employs a Perceiver-style cross-attention pooling layer. For each dynamically segmented patch, a query vector is initialized by max-pooling the constituent byte representations, followed by a linear projection.This query vector then attends exclusively to the byte hidden states within its respective patch boundary. This cross-attention pooling condenses variable-length byte information into a fixed-dimension latent space, allowing the global latent Transformer to operate across patches.Autoregressive Cross-Attention DecodingThe global latent Transformer maps the input patch representations $\mathbf{T}$ to updated latent representations $\mathbf{O} = [\mathbf{o}_1; \mathbf{o}_2; \dots; \mathbf{o}_M] \in \mathbb{R}^{M \times d_{\text{global}}}$. To convert these global latent states back into raw bytes, the local decoder utilizes $L_{\mathcal{D}}$ lightweight Transformer layers that alternate between cross-attention and causal local self-attention. At each decoder layer $l$, the byte-level hidden states $\mathbf{D}_l \in \mathbb{R}^{N \times d_{\text{local}}}$ are updated by attending directly to the global latent patch representations through cross-attention:$$\mathbf{B}_l = \mathbf{D}_{l-1} + \mathbf{W}_o \left( \text{softmax} \left( \frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}} \right) \mathbf{V} \right)$$where the projection states are defined as:$$\mathbf{Q}_i = \mathbf{W}_q(\mathbf{d}_{l-1, i})$$$$\mathbf{K}_j = \mathbf{W}_k(D_C(\mathbf{o}_j))$$$$\mathbf{V}_j = \mathbf{W}_v(D_C(\mathbf{o}_j))$$In these equations, $\mathbf{d}_{l-1, i}$ represents the individual byte hidden state, $\mathbf{o}_j$ is the latent global representation, and $D_C(\cdot)$ represents a linear transformation and dimension-splitting function.Following cross-attention, the updated representations $\mathbf{B}_l$ are processed by a standard causal Transformer layer using Rotary Position Embeddings (RoPE) to maintain local character alignment.The structural parameters and configuration of the standard BLT architecture are summarized in the table below:Hyperparameter / Configuration FieldArchitectural ComponentDefault Value / Specificationsvocab_sizeGlobal Patcher & Patcher260 (256 standard bytes + special tokens)patching_modeDynamic Segmentation"entropy"[cite: 12]patching_threshold ($\theta_g$)Dynamic Segmentation1.34max_position_embeddingsGlobal Latent Transformer4096cross_attn_kLocal Encoder Pooling2 (Number of cross-attention heads)encoder_hash_byte_group_sizeHash $n$-gram Embeddings$[3, 4, 5, 6, 7, 8]$ (Sliding window sizes)encoder_hash_byte_group_vocabHash $n$-gram Embeddings500,002 (Embedding table dimension)patcher_configPatcher (Entropy Model)100M parameters, 14 layers, 512 hidden sizeState Space Models and Linear-Time Sequence ComplexityTo address the quadratic scaling of standard attention mechanisms, selective State Space Models (SSMs)—specifically the Mamba family—have emerged as an alternative for long-sequence modeling. Inspired by classical continuous-time state space systems, these models map an input sequence $x(t) \in \mathbb{R}$ to an output sequence $y(t) \in \mathbb{R}$ through an implicit, high-dimensional latent state $h(t) \in \mathbb{R}^N$ using the following ordinary differential equations:$$h'(t) = \mathbf{A}h(t) + \mathbf{B}x(t)$$$$y(t) = \mathbf{C}h(t) + \mathbf{D}x(t)$$where $\mathbf{A}$, $\mathbf{B}$, $\mathbf{C}$, and $\mathbf{D}$ are structured transition matrices.To apply this continuous framework to digital sequences, the system is discretized using a step size parameter $\Delta$. Standard zero-order hold (ZOH) discretization maps the continuous transition parameters to their discrete counterparts:$$\mathbf{\bar{A}} = \exp(\Delta \mathbf{A})$$$$\mathbf{\bar{B}} = (\Delta \mathbf{A})^{-1}(\exp(\Delta \mathbf{A}) - \mathbf{I}) \cdot \Delta \mathbf{B}$$This discretization allows the system to be computed as a linear recurrence:$$h_t = \mathbf{\bar{A}}h_{t-1} + \mathbf{\bar{B}}x_t$$$$y_t = \mathbf{C}h_t + \mathbf{D}x_t$$The selective state space model (Mamba) parameterizes $\mathbf{B}$, $\mathbf{C}$, and $\Delta$ directly from the input sequence, allowing the model to selectively retain or discard information over time. Mamba-2 refines this formulation by introducing Structured State Space Duality (SSD), which establishes a theoretical link between state space recurrences and structured attention mechanisms. This duality allows SSD to be computed using highly optimized, hardware-friendly matrix multiplication kernels, significantly accelerating training and inference speeds.MambaByte: Token-Free Selective State Space ModelingThe linear sequence scaling of Mamba has been leveraged to build MambaByte, a token-free selective state space model trained autoregressively on raw byte streams. In contrast to standard Transformers, MambaByte compresses historical sequence context into a high-dimensional, fixed-size latent state that is independent of sequence length, maintaining linear scaling $\mathcal{O}(L)$ during training and constant-time execution during inference.In parameter-matched and compute-controlled benchmarks, MambaByte outperforms hierarchical architectures like MegaByte. For example, MambaByte-353M matches the performance of a MegaByte configuration (758M global parameters + 262M local parameters) while requiring $0.63\times$ less computational budget and training data.Additionally, MambaByte exhibits strong context length extrapolation, successfully processing sequence lengths up to $64\times$ longer than its training context without the catastrophic performance degradation typical of position-constrained Transformers.Crucially, MambaByte achieves this sequence modeling performance without relying on patch-based representation compression, which has been shown to degrade performance in traditional sequential architectures.Multimodal Adaptations: The Latent Speech-Text TransformerThe principles of dynamic latent sequence grouping also extend beyond text processing. The Latent Speech-Text Transformer (LST) applies these concepts to audio-text modeling. Traditional speech models rely on vector quantization to discretize continuous raw speech into uniform speech tokens, which can produce long, redundant sequences.LST addresses this by dynamically aggregating uniform speech tokens into variable-length latent patches. This dynamic compression concentrates compute on highly informative acoustic transitions while merging redundant frames, making multimodal pre-training more data-efficient.Hierarchical and Multi-Stage SSM-Transformer HybridizationTo scale byte-level modeling to exceptionally long sequences, researchers have developed hierarchical architectures that stack multiple sequence modeling layers. A key example is the Multiscale Byte Language Model (MBLM), which generalizes the two-stage hierarchy of models like MegaByte to an arbitrary number of stages ($N$ stages) and allows different model types, such as SSMs and Transformers, to be composed flexibly at different levels of the hierarchy.Hierarchical Formulation and Stage Parallelism in MBLMThe MBLM architecture consists of $N$ stacked causal decoder models, $M_{1}, M_{2}, \dots, M_{N}$. The first $N-1$ stages act as global inter-patch models, while the final stage $M_{N}$ serves as the local intra-patch model. Each stage operates on a specific hidden dimension $D_i \in \mathbb{N}$ and processes sequence patches of size $P_i \in \mathbb{R}$.The input byte sequence $x \in \mathbb{R}^{B \times L}$ is embedded for each stage $i$ and reshaped into a nested sequence of patch embeddings:$$x_{\text{emb}, i} \xrightarrow{\text{reshape}} \mathbf{P}_{\text{emb}, i} \in \mathbb{R}^{B \times P_1 \times \dots \times P_N \times D_N}$$To process these representations in parallel, the embeddings are packed into a flat batch dimension $K_i = B \cdot \prod_{j=1}^{i-1} P_j$ for all stages $i > 1$. This flattening allows the global models to perform inter-patch modeling. The outputs of a global stage $i$ are projected to the next stage $i+1$ using a linear projection layer $W^{\text{global}}_i$ and added to the target stage's embeddings:$$\mathbf{P}^{\text{in}}_i = \mathbf{P}^{\text{emb'}}_i + \mathbf{P}^{\text{out}}_{i-1} \mathbf{W}^{\text{global}}_{i-1}$$$$\mathbf{P}^{\text{out}}_i = \text{concat}_c M_i(\mathbf{P}^{\text{in}}_i)$$This multi-stage hierarchy allows the model to handle exceptionally long sequence lengths. For example, a 3D hybrid MBLM configuration—combining a global Mamba stage (Stage 1) with two subsequent Transformer stages (Stages 2 and 3)—successfully trained on context windows of up to 5 million bytes on a single 80GB GPU, achieving a bits-per-byte (BPB) rating of 2.448 on the Project Gutenberg test set.Adaptive Patching in Time-Series ForecastingThe principles of dynamic sequence compression have also been applied to non-textual modalities, such as time-series forecasting and physical system modeling. Architectures such as TimeSqueeze combine point-wise feature extraction with dynamic, content-aware patching.First, a lightweight selective SSM (Mamba) encoder extracts local, high-frequency temporal features at native resolution. Then, an adaptive patching module groups these embeddings into variable-sized patches, allocating smaller patches to information-rich, high-frequency regions and larger patches to redundant or flat regions.This dynamic compression reduces the input sequence length for the heavy downstream Transformer backbone, accelerating training speeds by up to $20\times$ and inference by $10\times$ while preserving critical structural details.This dynamic approach offers significant advantages over static patching frameworks like PatchTST, which rely on fixed patch boundaries and can struggle to capture irregular temporal shifts or sharp, localized transitions.Hierarchical Mamba with Hyperbolic GeometryAnother approach to modeling deep sequence hierarchies is the Hierarchical Mamba (HiM) framework, which integrates selective state space models with hyperbolic geometry. Standard neural architectures embed sequences in flat Euclidean spaces, which can struggle to represent complex, branching conceptual hierarchies.To address this, HiM projects Mamba2-processed sequence states onto a Poincaré ball ($\mathbb{D}^d_c$) or a Lorentzian manifold ($\mathbb{L}^d_c$) with a learnable curvature $c$. Euclidean outputs $\mathbf{z} \in \mathbb{R}^d$ are projected onto the Poincaré ball via tangent-space exponential mapping:$$\text{proj}(\mathbf{z}) = \tanh(\sqrt{c}\Vert{}\mathbf{z}\Vert{})\frac{\mathbf{z}}{\sqrt{c}\Vert{}\mathbf{z}\Vert{}}$$By mapping these latent sequence states to a hyperbolic manifold, HiM can represent complex hierarchical structures and taxonomic relations.In evaluation, HiM-Poincaré captures fine-grained semantic distinctions with high hyperbolic norms, while HiM-Lorentz provides stable, compact, and hierarchy-preserving embeddings. This architecture enables efficient long-range reasoning and hierarchical classification across medical ontologies and structured linguistic ontologies.Inference Acceleration and Parallel Generation ArchitecturesWhile dynamic latent patch representations and selective state space models reduce the computational complexity of byte-level architectures, sequential generation remains a primary inference bottleneck. In modern hardware, generating text byte-by-byte is highly inefficient because it repeatedly reloads heavy model parameters and key-value (KV) caches from High Bandwidth Memory (HBM).To address this memory-bandwidth bottleneck, researchers have developed parallel decoding and speculative verification techniques.Speculative Subword Drafting and Parallel VerificationTo accelerate inference in raw byte models, MambaByte implements a speculative decoding pipeline that pairs a fast, tokenized subword draft model ($M_{\text{subword}}$) with the token-free byte-level verifier ($M_{\text{byte}}$).Drafting Step:

The drafting and verification pipeline operates through two alternating steps:Drafting Step: A smaller subword Mamba model generates a sequence of candidate subwords, recording its internal states.Verification Step: The candidate subwords are decoded into raw bytes and processed in parallel by the MambaByte verifier model. Utilizing the parallel associative scan of the selective SSM, the verifier computes next-step probabilities across the entire draft sequence in a single parallel pass.During verification, the system identifies the furthest byte position $c$ that is verified to fall within the top-$\beta$ autoregressive candidates of the MambaByte model. All drafted bytes occurring after this bifurcation point $c$ are discarded, and the model rolls back to that position.To maintain alignment and avoid out-of-vocabulary (OOV) errors in the draft model's tokenizer, the byte-level verifier temporarily takes over generation, producing bytes autoregressively until a boundary byte (such as a space) is created.To optimize this process, MambaByte utilizes a custom CUDA kernel that allows the parallel verification scan to resume directly from the mismatched bifurcation position. This pipeline delivers up to a $2.6\times$ inference speedup over standard MambaByte implementations, bringing its generation speed in line with subword-based models.Fast BLT VariantsTo mitigate the memory-bandwidth bottleneck in the Byte Latent Transformer, three inference techniques have been developed to generate multiple bytes in parallel per decoding step.BLT Diffusion (BLT-D)BLT Diffusion (BLT-D) accelerates generation by replacing byte-by-byte autoregressive decoding with parallel block-wise discrete diffusion. During training, the local decoder is trained with a dual objective: the standard next-byte prediction loss on clean sequences, and a masked-byte prediction loss on corrupted byte blocks of fixed length.These corrupted blocks are constructed from dynamically segmented patches but are allowed to extend beyond individual patch boundaries, teaching the decoder to predict future sequences beyond average patch lengths.During generation, the decoder initializes a block of future byte positions as [MASK] tokens. It then generates multiple bytes in parallel via semi-autoregressive text diffusion, conditioning on the latent representation from the global model.By iteratively unmasking and predicting multiple byte positions in parallel, BLT-D reduces the number of forward passes required by the global model.For example, the BLT-D-16 variant (which processes blocks of 16 bytes) achieves an 87% to 92% memory-bandwidth reduction compared to standard BLT, with only minor tradeoffs in task performance.BLT Self-Speculation (BLT-S)BLT Self-Speculation (BLT-S) accelerates generation without requiring model retraining or architectural modifications. In standard BLT, the local decoder stops generating as soon as the entropy patcher identifies a patch boundary, triggering a call to the global model.BLT-S bypasses this constraint by allowing the lightweight local decoder to continue generating (drafting) several bytes autoregressively past standard patch boundaries.The full global model then verifies the drafted sequence in a single forward pass. Drafted bytes are accepted up to the first mismatch; if a mismatch is found, generation rolls back to that position, and the correct byte is inserted.Under greedy decoding, this speculative process is mathematically guaranteed to produce outputs bit-for-bit identical to standard autoregressive decoding. This method reduces memory-bandwidth consumption by up to 77% with zero loss in generation quality.BLT Diffusion and Verification (BLT-DV)BLT Diffusion and Verification (BLT-DV) combines the speed of block-wise diffusion drafting with the quality guarantees of speculative verification, using a single set of model weights.The model first generates a candidate block of future bytes in parallel using the fast diffusion decoder. It then runs a single causal verification forward pass through the global model and decoder.The drafted bytes are verified against the model's next-byte predictions, accepting them up to the first mismatch.This hybrid pipeline limits the quality degradation typical of pure diffusion models while achieving up to an 81% reduction in memory-bandwidth cost compared to standard autoregressive BLT.Domain-Specific Byte Structuring and SSM ApplicationsOperating directly on raw bytes with selective state space models has enabled new applications across a variety of domains. tokenizer-free foundation models. By eliminating static, language-biased subword vocabularies, byte-level modeling simplifies the processing of low-resource languages, handles character-level noise, and provides a unified interface for highly diverse data modalities.By combining the linear computational complexity of selective State Space Models (Mamba-2) with the content-aware patch allocation of architectures like the Byte Latent Transformer (BLT), modern models can process extended context windows without the quadratic computational bottleneck of traditional attention mechanisms.The high latency of sequential, byte-by-byte autoregressive decoding remains a primary inference bottleneck, but it is being addressed through parallel generation techniques. By processing data at the fundamental level of raw bytes, these architectures are well-positioned to serve as the foundation for unified, omni-modal AI systems capable of processing text, programming code, network traffic, hyperspectral data, and raw binary formats within a single, cohesive framework.
